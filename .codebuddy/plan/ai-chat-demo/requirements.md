# 需求文档：AI Chat 浮窗与智能场景演示

## 引言

mindX 作为用户使用 Agent 的外挂大脑，用户的交互出入口在 Agent 软件（如 WorkBuddy）上。为了在 Demo 中展示这一核心交互模式，需要在页面上增加一个轻量级的 AI Chat 浮窗组件，模拟用户通过自然语言与 Agent 交互的场景。

本次迭代包含三个核心部分：

1. **AI Chat 浮窗组件**：右下角悬浮按钮 + 可展开/收起的聊天窗口，预设指令和 Mock 回复
2. **智能分析报告生成场景**：用户通过 AI Chat 触发分析任务，系统生成一份基于现有 Mock 数据的分析报告文档，报告中通过双向链接溯源引用原始数据
3. **智能表格数据操作场景**：用户通过 AI Chat 自然语言指令新增客户数据到 CRM 表格，展示 Agent 操作结构化数据的能力

这两个场景将展示 mindX 作为 Agent 外挂大脑的核心价值：**智能分析决策**和**自然语言操作数据**的能力。

---

## 需求

### 需求 1：AI Chat 浮窗组件

**用户故事：** 作为一名 Demo 演示者，我希望页面右下角有一个可展开/收起的 AI Chat 浮窗，以便我能模拟用户通过 Agent 软件与 mindX 交互的场景。

#### 验收标准

1. WHEN 用户进入任意页面 THEN 系统 SHALL 在页面右下角显示一个悬浮的 AI Chat 按钮（类似 Notion AI 的交互风格）
2. WHEN 用户点击悬浮按钮 THEN 系统 SHALL 展开一个聊天窗口浮层，窗口包含：消息列表区域、预设指令按钮区域、输入框区域
3. WHEN 聊天窗口已展开且用户点击关闭按钮或悬浮按钮 THEN 系统 SHALL 收起聊天窗口，仅保留悬浮按钮
4. WHEN 聊天窗口展开 THEN 系统 SHALL 在输入框上方显示预设的常用指令按钮（快捷指令 chips），用户可一键点击发送
5. WHEN 用户点击预设指令按钮或在输入框中输入文本并发送 THEN 系统 SHALL 将用户消息显示在消息列表中，并在短暂延迟（模拟思考）后显示对应的 Mock 回复
6. WHEN Mock 回复中包含文档或表格链接 THEN 系统 SHALL 将链接渲染为可点击的蓝色超链接，点击后通过路由跳转到对应的文档/表格页面
7. WHEN 用户点击清空按钮 THEN 系统 SHALL 清空当前对话历史
8. IF 用户输入的文本不匹配任何预设指令 THEN 系统 SHALL 显示一条通用的 Mock 回复（如"这个功能正在开发中，敬请期待"）
9. WHEN AI Chat 浮窗渲染 THEN 系统 SHALL 确保浮窗样式与现有系统保持一致（使用项目现有的 inline style 风格，不引入外部 UI 库）

### 需求 2：智能分析报告生成场景

**用户故事：** 作为一名 Demo 演示者，我希望通过 AI Chat 发送分析指令后，系统能自动生成一份基于现有数据的分析报告文档，以便展示 Agent 智能分析决策的能力。

#### 验收标准

1. WHEN 用户在 AI Chat 中发送预设指令"分析当前库存状况和供应链风险" THEN 系统 SHALL 显示 Mock 回复："已为您生成库存与供应链分析报告，点击查看详细分析 → 📊 库存与供应链分析报告-20250410"，其中报告名称为可点击的链接
2. WHEN 分析报告生成 THEN 系统 SHALL 在文档列表（`initialDocuments`）中动态添加一个新的 Smart Canvas 文档，文档 ID 为 `canvas-analysis-report`，标题为"库存与供应链分析报告-20250410"
3. WHEN 用户点击 Mock 回复中的报告链接 THEN 系统 SHALL 跳转到该报告文档的编辑/查看页面
4. WHEN 报告文档渲染 THEN 文档内容 SHALL 包含以下章节：
   - 4.1 报告概要（生成时间、数据范围说明）
   - 4.2 库存状况分析（基于 `sheet-inventory` 库存表数据，包含各 SKU 库存水位、安全线对比）
   - 4.3 供应链风险评估（基于 `sheet-suppliers` 供应商表数据，包含供应商交期、产能评估）
   - 4.4 客户需求影响分析（基于 `sheet-crm` 客户表和 `sheet-orders` 订单表数据）
   - 4.5 业务建议与行动项
5. WHEN 报告文档中引用原始数据 THEN 系统 SHALL 使用 `ref` 注解（双向链接）标注数据来源，包括：
   - 5.1 引用库存表中的具体 SKU 行记录（如 `@经典款陶瓷马克杯`）
   - 5.2 引用供应商表中的供应商行记录（如 `@景德镇宏图陶瓷`）
   - 5.3 引用已有的 Smart Canvas 文档（如 `@Q2 华东区供应链异常预警报告`、`@重点客户 Q1 拜访纪要`）
6. WHEN 报告文档创建 THEN `crossReferences.ts` 中 SHALL 同步新增对应的 CrossReference 记录，确保双向链接图谱完整
7. WHEN 报告文档中的 `@引用` 指向表格行记录 THEN 系统 SHALL 复用已有的行展开浮窗交互（点击弹出 RowExpandModal）
8. WHEN 报告文档中的 `@引用` 指向其他 Canvas 文档 THEN 系统 SHALL 通过路由跳转到目标文档

### 需求 3：智能表格数据操作场景

**用户故事：** 作为一名 Demo 演示者，我希望通过 AI Chat 发送自然语言指令后，系统能自动在对应表格中新增一行数据，以便展示 Agent 自然语言操作结构化数据的能力。

#### 验收标准

1. WHEN 用户在 AI Chat 中发送预设指令"新增客户：腾讯集团 shrekyan，深圳南山区腾讯总部，主营企业定制礼品采购，年采购预算 500 万，对接人 Shrek Yan，联系方式 shrekyan@tencent.com" THEN 系统 SHALL 显示 Mock 回复："已将新客户「腾讯集团-shrekyan」添加到 B2B 客户档案与授信表 → 📋 B2B 客户档案与授信表"，其中表格名称为可点击的链接
2. WHEN 新增客户指令执行 THEN 系统 SHALL 在 `sheet-crm`（B2B 客户档案与授信表）中动态添加一行新数据，行 ID 为 `row-shrekyan`，包含以下字段：
   - 2.1 客户名称：`腾讯集团-shrekyan`
   - 2.2 客户评级：`A 级`（新客户默认评级）
   - 2.3 账期与折扣：`月结 45 天，基准价 95 折`
   - 2.4 历史客诉状态：`无异常`
   - 2.5 关联上下文：关联到分析报告文档（如有）
3. WHEN 新增行数据创建 THEN 该行 SHALL 包含 `expandBlocks`（行展开内容），展示客户的详细描述信息，包括：
   - 3.1 客户背景介绍（腾讯集团企业采购部门，Shrek Yan 为采购负责人）
   - 3.2 采购需求描述（企业定制礼品、员工福利、客户赠品等场景）
   - 3.3 初始合作计划（试采阶段的 SKU 推荐、报价方案等）
4. WHEN 用户点击 Mock 回复中的表格链接 THEN 系统 SHALL 跳转到 CRM 表格页面，且新增的行应可见
5. WHEN 新增行在表格中显示 THEN 用户 SHALL 能够点击该行展开查看详细信息（复用已有的 RowExpandModal）
6. WHEN 新增客户数据添加到表格 THEN 系统 SHALL 确保数据格式与现有行数据一致，不破坏表格渲染

### 需求 4：AI Chat 预设指令与 Mock 数据管理

**用户故事：** 作为一名开发者，我希望 AI Chat 的预设指令和 Mock 回复数据集中管理，以便后续方便扩展新的演示场景。

#### 验收标准

1. WHEN 开发者需要新增预设指令 THEN 系统 SHALL 提供一个集中的数据配置文件（如 `aiChatMockData.ts`），包含所有预设指令、匹配规则和 Mock 回复内容
2. WHEN 预设指令配置 THEN 每条指令 SHALL 包含：指令文本、快捷按钮显示文本、Mock 回复内容、可选的副作用动作（如创建文档、添加表格行）
3. WHEN Mock 回复中包含副作用动作 THEN 系统 SHALL 在显示回复的同时执行对应的数据变更操作（如向 `documents` 状态中添加新文档、向 `sheetData.rows` 中添加新行）
4. WHEN 系统执行副作用动作 THEN 数据变更 SHALL 通过 `MindXDemoContext` 的状态管理机制进行，确保 UI 响应式更新

## 技术约束与边界

- **轻量实现**：AI Chat 浮窗仅用于 Demo 演示，不需要真实的 AI 后端，所有回复均为预设的 Mock 数据
- **样式一致性**：使用项目现有的 inline style 风格，不引入外部 UI 组件库，与现有系统视觉风格保持一致
- **状态管理**：新增文档和表格行数据通过 `MindXDemoContext`（React Context + useState）管理，确保全局状态同步
- **路由兼容**：AI Chat 浮窗需要在所有页面上可见（全局组件），生成的文档/表格链接需要正确路由到对应页面
- **数据完整性**：新增的 Mock 数据（分析报告文档、新客户行数据）需要与现有数据结构完全兼容，包括类型定义、ref 注解格式（`[["ref", ...]]` 双层数组）等
- **组件复用**：报告文档中的 `@引用` 交互复用已有的行展开浮窗和文档跳转逻辑，不新建交互模式
- **无外部依赖**：所有改动使用原生 React + TypeScript，与项目现有技术栈一致
- **浮窗层级**：AI Chat 浮窗的 z-index 需要高于页面内容但低于 Modal 弹窗，避免遮挡行展开浮窗等交互
