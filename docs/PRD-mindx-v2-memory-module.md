# MindX 2.0 Memory 模块需求文档

> **版本**：v0.9  
> **日期**：2026-04-01  
> **状态**：Current Prototype Definition  
> **产品**：[[MindX]] / [[MindX 2.0]] / [[Memory]]

---

## 一、文档目的

这不是一个“未来理想态”PRD。

这份文档的作用是把 **2026-04-01 上午这一轮已经落到原型里的产品定义** 固定下来，避免后续页面继续迭代时又把边界改乱。

它要回答 4 个问题：

1. [[MindX 2.0]] 和 [[MindX 1.0]] 到底是什么关系  
2. `Workspace / Memory / Integrations` 这三块现在怎么分工  
3. `Memory` 模块内部到底包含哪些页面、哪些不再单独暴露  
4. `数据源 / 知识资产 / Timeline / 编辑文档 / 接入记忆` 这些动作之间的真实关系是什么

---

## 二、产品关系

### 1. 2.0 不是新产品

[[MindX 2.0]] 是 [[MindX 1.0]] 的升级版，不是另一款产品。

当前保留 `切换到 2.0`，只是为了让 1.0 和 2.0 在原型阶段并行跑，方便持续迭代和对照验收。  
从产品叙事上，用户应该感受到的是：

- 还是同一个 [[MindX]]
- 只是 `Memory` 明显变强了
- `Workspace` 仍然是主工作台

### 2. 1.0 当前真实结构

1.0 当前侧边栏分组为：

- `Workspace`
- `Skills`
- `Memory`

其中 1.0 的 `Memory` 已经真实存在，但它只承接 **基础长期记忆**，不是 2.0 的知识资产系统。

1.0 当前 `Memory` 的真实内容更接近：

- `Who am I`
- `Directives`
- `Goals`
- 原始记忆文档

它是“长期上下文底座”，不是“记忆卡片产品”。

---

## 三、2.0 的当前定义

### 1. 顶层导航

2.0 当前顶层分组固定为：

1. `Workspace`
2. `Memory`
3. `Integrations`

这里有两个硬边界：

- `Knowledge` 不是一级模块
- `Timeline` 也不是一级模块

### 2. 默认入口

2.0 默认从 `Workspace` 进入，而不是直接落到 `Memory`。

原因很简单：

- 用户对 [[MindX]] 的第一认知仍然是工作区
- `Memory` 必须长在工作流里，而不是漂在工作区外面
- 2.0 的增强应该建立在 1.0 已有工作台之上

---

## 四、模块边界

### 1. Workspace

`Workspace` 继续承接 1.0 的文档主视图。

当前要求是：

- 除新增 `Memory` 关联列外，整体列表结构、筛选逻辑、打开文档方式都尽量与 1.0 保持一致
- `Workspace` 依然是人和 agent 协作发生的地方
- `Workspace` 不是原始记忆资料库
- 从 `Memory` 接入的外部资料、上传文档、同步记录 **不应直接进入 Workspace 文档列表**

`Workspace` 在 2.0 中只允许做“弱关联”：

- 从文档跳到相关记忆
- 从记忆回到来源文档

不允许把它改造成资产页或知识后台。

### 2. Memory

`Memory` 是 2.0 本期唯一做满的模块。

它当前包含 4 类内容：

1. **基础记忆**
   - 继承 1.0 已有内容
   - 包括 `关于我 / 写作原则 / 我的目标`
2. **Timeline**
   - 表达“记忆正在形成”
   - 不是侧边栏单独模块，而是从 `Memory` 首页的“正在生成 / 查看详情”进入
3. **Knowledge**
   - 表达已经沉淀的知识资产、标签分类、AI 察觉
4. **数据源**
   - 存放 Memory 接入进来的原始资料
   - 包括上传文件、聊天记录、网页剪存、云端同步、插件同步

### 3. Integrations

2.0 的第三组不再叫 `Skills`，而是 `Integrations`。

它承接的是：

- 外部 agent / AI 应用接入
- 已挂载的记忆包
- 读写与回写闭环

也就是说：

- `Knowledge` 管资产本体
- `Integrations` 管这些资产如何被挂载、读取、回写

---

## 五、Memory 的内部结构

### 1. Memory 首页

当前 `Memory` 首页是一个入口页，而不是说明页。

页面结构固定为：

1. 顶部三张核心卡片
   - `基础记忆`
   - `我的目标`
   - `Knowledge`
2. 左下 `问 AI`
   - 使用 `memo agent`
   - 用户可以直接提问：检索、溯源、洞察、判断哪些记忆值得挂载
3. 右下 `正在生成`
   - 展示近期正在形成的记忆线索
   - 提供 `查看详情` 跳去 `Timeline`

页面里不应该出现大段解释“2.0 是什么”的说明文案。

### 2. Timeline

`Timeline` 是 `Memory` 的内部明细页，不在左侧导航单独出现。

它的职责是：

- 展示一天里发生了什么
- 这些事如何逐步沉淀成记忆
- 哪些线索还在形成，哪些已经沉淀

当前进入方式：

- 从 `Memory` 首页右侧“正在生成”点 `查看详情`
- 从 `Workspace` 文档里的记忆弱关联入口进入相关 timeline 上下文

### 3. Knowledge

`Knowledge` 是 `Memory` 内部的结构化资产页。

当前大结构已经固定，不再改：

- `资产库`
- `标签分类`
- `AI 察觉`

这一层是资产整理层，不是原始资料层，也不是新入口层。

### 4. 数据源

`数据源` 是这轮新补齐的 Memory 子模块。

它存在的原因是：

- 接入到 Memory 的资料不一定是协作文档
- 这些原始资料不该混入 `Workspace`
- 资产和原始资料必须分层

因此当前明确区分：

- `数据源`：原始资料与外部接入内容
- `Knowledge`：沉淀后的结构化资产

当前 `数据源` 页面要求：

- 用接近 1.0 `Workspace` 文档页的高密度列表样式
- 顶部有来源分类筛选
- 支持点击进入文档视图查看原始内容

当前侧边栏顺序固定为：

1. `Memory`
2. `Knowledge`
3. `数据源`

## 六、核心动作定义

### 1. Workspace：新建文档

`Workspace` 顶部动作统一为：

- `新建文档`

它创建的是协作文档，并直接进入文档编辑页。

### 2. Memory：接入记忆

`Memory` 顶部动作统一为：

- `接入记忆`

这个动作不再创建 Workspace 文档，而是把外部内容接入到 `数据源`。

当前接入弹层按来源分组展示：

- 通用文档
- 搜索与收藏
- 云端协作
- 第三方插件同步

它代表的是“把资料带进 Memory”，不是“新建协作文档”。

### 3. Integrations：新建集成

`Integrations` 顶部动作统一为：

- `新建集成`

它代表：

- 新接入 agent / 应用
- 新建挂载关系
- 新建回写规则

---

## 七、知识资产与文档的关系

### 1. 知识卡片只是预览，不是终点

`Knowledge` 里的卡片预览只是资产化摘要。

它背后对应的仍然是一篇可以继续处理的文档化内容。  
因此当前产品行为必须支持两条路：

1. **基于卡片和 AI 对话**
   - 通过 `memo agent` 围绕这张卡继续追问、补标签、找关联、优化说明
2. **直接编辑文档**
   - 点击 `编辑文档`
   - 进入文档视图
   - 用 1.0 现有的文档交互继续处理，包括划词评论

### 2. memo agent 在资产详情中的职责

资产详情里的 `memo agent` 当前职责是：

- 溯源证据
- 优化说明
- 补充标签
- 寻找关联
- 帮用户决定是否进一步改写这张资产

它不是单独的新页面，而是嵌在资产详情里的辅助处理器。

---

## 八、数据流定义

### 1. Workspace 文档流

`Workspace` 文档是协作空间内的文档资产。

它们：

- 可以被人和 agent 共同编辑
- 可以从文档页跳向相关记忆
- 但并不自动等于 Memory 数据源

### 2. Memory 数据源流

从 `接入记忆` 进入的内容，当前统一走：

`接入记忆 -> 数据源 -> 可被提炼 -> 进入 Timeline / Knowledge`

它们：

- 默认进入 `数据源`
- 可以在文档视图中查看和编辑
- 但不会直接塞进 `Workspace` 文档列表

### 3. Knowledge 资产流

`Knowledge` 里的卡片来自于：

- Workspace 协作文档里的记忆线索
- Timeline 中已形成的阶段性判断
- 数据源中被提炼出来的结构化内容

因此它是“沉淀后的结果层”，不是“原始输入层”。

## 九、当前原型验收标准

### 阻塞级

以下任一项不满足，都视为当前原型不可验收：

1. `切换到 2.0` 后默认没有进入 `Workspace`
2. 2.0 顶层结构不是 `Workspace / Memory / Integrations`
3. `Knowledge` 跑到一级导航
4. `Timeline` 在左侧成为单独模块
5. `接入记忆` 的内容进入了 `Workspace`
6. `数据源` 不存在独立承接页
7. `编辑文档` 不能从知识卡详情真正进入文档视图
8. `Workspace` 的文档打开方式与 1.0 明显不一致

### 通过级

以下项目应能稳定演示：

1. 用户从 1.0 切换到 2.0，仍觉得是同一个产品
2. 用户能理解 `Workspace` 是协作空间、`Memory` 是沉淀层、`Integrations` 是挂载层
3. 用户能在 `Memory` 首页看到：
   - 三张基础记忆卡
   - `memo agent`
   - `正在生成`
4. 用户能从 `正在生成` 进入 `Timeline`
5. 用户能从 `Knowledge` 卡片预览里：
   - 与 `memo agent` 继续对话
   - 进入文档编辑
6. 用户能在 `数据源` 里看到 Memory 接入进来的原始资料，且它们不混进 Workspace

---

## 十、本期明确不做

本轮明确不做这些事情：

- 不把 2.0 做成另一款产品
- 不重构 Workspace 主逻辑
- 不把 Timeline 单独升成一级模块
- 不把 Knowledge 升成一级模块
- 不做组织级 / 团队级 memory 体系
- 不做生产级后端记忆引擎

本轮交付物始终是：

**可演示、可讲清、可继续迭代的产品原型。**
