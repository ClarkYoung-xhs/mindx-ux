# 问题：Memory 1.0 被做成了“混合设计”而不是直接复刻 2.0 骨架

## what
这次改造一开始没有直接把 2.0 的展示骨架搬到 1.0 上，而是做了一个介于两者之间的中间态。结果是模块映射虽然对了，但视觉结构、状态表达和文案节奏都还混着 1.0 和 2.0 的语言。

## why
根因不是实现能力不够，而是产品判断不够硬。
当用户已经明确给出 1.0 -> 2.0 的模块映射时，正确动作应该是直接复刻 2.0 的展示骨架，再把 1.0 数据接上去，而不是继续“参考 2.0”然后自己发明过渡版。

## how
- 纯展示层模块先看 2.0 的信息架构和卡片骨架
- 先复制结构和节奏，再接入 1.0 的数据和状态
- 只保留真实状态，示例和说明性文案只能出现在明确空态
- 已完成、已提炼、知识摘要这类区块，不要再出现进行态词汇

## action
- 后续 Memory UI 只允许“2.0 骨架 + 1.0 数据”的实现方式
- 任何新增文案先过一遍“这是不是示例 / 解释 / 进行态”
- 验收时必须检查重复入口、状态 badge、示例卡片、说明性占位词

## guardrail
- 不允许把“参考 2.0”当成交付标准，必须明确到“直接复刻骨架”
- 不允许在生产 Memory UI 里出现 `示例`、`sample`、`正在生成`、`正在收敛` 这类词，除非它真的是临时空态
- 只要一个 section 标题是“已提炼/已完成”，里面就不能再出现进行态视觉和文案
