# MindX 登录模块 1.0 需求文档

> **版本**：v2.2 | **日期**：2026-03-31 | **状态**：Draft  
> **产品**：MindX 协作空间 | **模块**：登录与注册

---

## 一、目标 / 背景说明

### 1. 需求背景

MindX 是面向人类和 AI Agent 的协作文档空间。登录模块是用户进入产品的第一触点，直接影响注册转化率和首次体验。

当前仅支持邮箱验证码登录，步骤多、缺少社交登录、无状态持久化，海外用户习惯 Google 一键登录却找不到入口。

### 2. 数据目标

**规模类**

| 指标 | 基线 | 目标 |
|------|------|------|
| 登录转化率（Landing → Dashboard） | — | ≥ 75% |
| 第三方登录占比 | 0% | ≥ 40% |

**健康度**

| 指标 | 基线 | 目标 |
|------|------|------|
| 7 日登录留存 | — | ≥ 50% |
| 登录失败率 | — | ≤ 5% |

**性能类**

| 指标 | 目标 |
|------|------|
| 邮箱路径平均登录时长 | ≤ 30s |
| Google 路径平均登录时长 | ≤ 8s |
| 验证码接口响应 | ≤ 3s |

### 3. 竞品参考

| 竞品 | 邮箱 | Google | Facebook | 特点 |
|------|------|--------|----------|------|
| Notion | Magic Link | ✅ | ❌ | 无密码，纯链接 |
| Linear | 验证码 | ✅ | ❌ | 极简弹窗 |
| Figma | 密码 | ✅ | ✅ | SSO + 多方式 |
| **MindX** | **验证码** | **✅** | **✅** | **弹窗 + 多语言** |

### 4. 实施计划

| 阶段 | 内容 | 周期 |
|------|------|------|
| Phase 1 | 邮箱验证码 + 状态管理 + 多语言 | 1 周 |
| Phase 2 | Google OAuth | 1 周 |
| Phase 3 | Facebook OAuth | 0.5 周 |
| Phase 4 | JWT Token + 安全加固 | 1 周 |
| Phase 5 | 埋点 + 监控 | 0.5 周 |

---

## 二、灰度策略

- Phase 1（邮箱验证码）：全量发布
- Phase 2-3（OAuth）：10% → 50% → 100%，每批观察 3 天
- 回退条件：OAuth 失败率 > 10% 时关闭入口

---

## 三、适配范围

### 本期做

- 邮箱验证码登录（发送 + 验证 + 倒计时 + 返回修改）
- Google / Facebook OAuth 登录
- 快捷登录（测试环境）
- 登录状态持久化
- 中英文双语
- 弹窗式登录
- 首次登录 → Onboarding 衔接

### 明确不做

- 手机号短信登录、密码登录
- Apple ID / 微信 / QQ
- 2FA、SSO/SAML
- 账号合并、用户中心
- 登录频率限制（Phase 4 补充）

### 产品决策记录

| 决策 | 结论 | 理由 |
|------|------|------|
| 弹窗 vs 页面跳转 | 弹窗 | 不打断浏览，竞品 Notion/Linear 同样做法 |
| 外部点击不关闭 | 是 | 防止误触丢失已输入内容 |
| 去掉「登录」入口 | 是 | 只保留「开始使用」，减少选择焦虑 |
| 先不做密码 | 是 | 目标用户习惯验证码/社交登录 |
| 验证码 vs Magic Link | 验证码 | Magic Link 可能在不同浏览器打开导致状态丢失 |

### 4.3 异常场景

| 场景 | 系统响应 | 用户可操作 |
|------|---------|-----------|
| 邮箱格式错误 | 红色提示"请输入有效的邮箱地址" | 修改后重试 |
| 验证码输入错误 | 红色提示"验证码错误" | 重新输入 |
| 验证码过期（>5min） | 提示"验证码已过期" | 点击重新发送 |
| 60s 内重复发送 | 按钮置灰，显示倒计时 | 等待 |
| 24h 发送超 10 次 | 提示"发送次数过多，请稍后再试" | 等待或换登录方式 |
| 网络断开 | 提示"网络异常，请检查后重试" | 恢复网络后重试 |
| OAuth 取消授权 | 返回登录弹窗，不报错 | 选择其他方式 |
| OAuth 超时/服务不可用 | 提示"服务暂时不可用，请使用邮箱登录" | 切换邮箱登录 |

## 五、功能详细说明

### （1）登录入口

| 操作 | 结果 | 异常处理 |
|------|------|---------|
| 用户点击导航栏「开始使用」 | 弹出登录弹窗，背景显示半透明遮罩 | — |
| 用户点击弹窗外部区域 | 无反应，弹窗不关闭 | — |
| 用户点击右上角 X 按钮 | 弹窗关闭，回到 Landing Page | — |
| 已登录用户打开 mindx.app | 自动跳转 Dashboard，不展示 Landing Page | 登录态异常/过期：清除状态，展示 Landing Page |

说明：导航栏不单独显示「登录」入口，仅保留「开始使用」一个按钮。

### （2）邮箱验证码登录

**步骤 1 — 输入邮箱**

| 操作 | 结果 | 异常处理 |
|------|------|---------|
| 用户在输入框输入邮箱 | 实时校验格式，输入框左侧显示邮箱图标 | — |
| 用户输入合法邮箱，点击「继续」 | 按钮变为 Loading 状态（旋转图标 + "发送中..."），约 1-3s 后跳到步骤 2 | 网络异常：显示"网络异常，请检查后重试"，按钮恢复可点击 |
| 用户输入非法邮箱（如 "abc"），点击「继续」 | 输入框下方显示红色提示"请输入有效的邮箱地址" | — |
| 用户邮箱为空时查看「继续」按钮 | 按钮置灰不可点击 | — |
| 用户在输入框按 Enter | 等同于点击「继续」按钮 | — |
| 同一邮箱 24h 内发送超过 10 次 | 显示"发送次数过多，请稍后再试" | 引导用户使用第三方登录 |

**步骤 2 — 输入验证码**

| 操作 | 结果 | 异常处理 |
|------|------|---------|
| 页面展示 | 标题变为"检查你的邮箱"，副标题"验证码已发送至 xxx@xxx.com"，顶部进度条两段全亮 | — |
| 用户输入 6 位正确验证码 | 验证成功，新用户跳转 Dashboard + Onboarding，老用户直接 Dashboard | — |
| 用户输入错误验证码 | 红色提示"验证码错误，请重试"，输入框内容保留，可修改后重试 | — |
| 用户输入非数字字符 | 自动过滤，只保留数字 | — |
| 用户输入不满 6 位时查看「验证」按钮 | 按钮置灰不可点击 | — |
| 用户在输入框按 Enter（已满 6 位） | 等同于点击「验证」按钮 | — |
| 验证码超过 5 分钟后输入 | 红色提示"验证码已过期，请重新发送" | 用户点重发获取新验证码 |
| 用户点击「重新发送」（倒计时结束后） | 重新发送验证码，倒计时重置为 60s | 网络异常：显示错误提示 |
| 用户在 60s 倒计时内查看重发按钮 | 按钮置灰，显示"X 秒后可重新发送" | — |
| 用户点击「返回」 | 回到步骤 1，邮箱保留，验证码和错误提示清空 | — |

### （3）第三方社交登录

**Google 登录**

| 操作 | 结果 | 异常处理 |
|------|------|---------|
| 用户点击「使用 Google 登录」 | 跳转 Google OAuth 授权页 | Google 服务不可用：提示"服务暂时不可用，请使用邮箱登录" |
| 用户在 Google 页面选择账号并授权 | 回调 MindX，自动完成登录，跳转 Dashboard | 回调参数异常（state 不匹配）：静默失败，返回登录弹窗 |
| 用户在 Google 页面点击取消 | 返回登录弹窗（邮箱输入页），不显示任何错误 | — |
| Google 授权超时（>30s） | 提示"登录超时，请重试" | — |
| Google 返回的邮箱已有 MindX 账号 | 直接登录该账号 | — |

**Facebook 登录**

| 操作 | 结果 | 异常处理 |
|------|------|---------|
| 用户点击「使用 Facebook 登录」 | 跳转 Facebook OAuth 授权页 | Facebook 服务不可用：提示"服务暂时不可用，请使用邮箱登录" |
| 用户授权成功 | 回调 MindX，自动完成登录 | — |
| 用户取消授权 | 返回登录弹窗，不报错 | — |

**快捷登录（测试环境）**

| 操作 | 结果 | 异常处理 |
|------|------|---------|
| 用户点击「⚡ 快捷登录（测试）」 | 直接以 demo 账号登录，跳转 Dashboard | — |

说明：三个按钮位于「── 或 ──」分隔线下方，排列顺序为 快捷登录 → Google → Facebook。

### （4）登录状态管理

| 场景 | 系统行为 | 异常处理 |
|------|---------|---------|
| 任意方式登录成功 | 写入登录态到 localStorage，记录用户邮箱 | — |
| 首次登录成功 | 跳转 `/dashboard?onboarding=true`，触发 Onboarding 引导 | — |
| 回访用户打开网站 | 页面加载时检测 localStorage，有值则直接跳 Dashboard | 登录态数据损坏：清除并展示 Landing Page |
| 用户点击「登出」 | 清除 localStorage → 跳转 Landing Page | — |

说明：Phase 4 将 localStorage 迁移到 JWT + HttpOnly Cookie，7 天有效，支持服务端校验和多端同步。

### （5）多语言

| 场景 | 结果 |
|------|------|
| Landing Page 当前语言为中文 | 登录弹窗所有文案显示中文 |
| Landing Page 当前语言为英文 | 登录弹窗所有文案显示英文 |
| 弹窗打开后切换语言 | 弹窗内文案实时跟随切换 |

覆盖文案：标题、副标题、输入框标签和 placeholder、按钮文案、错误提示、倒计时文案、分隔线「或/OR」、第三方按钮文案、服务条款提示。

## 七、依赖与约束

### 依赖项

| 依赖 | 说明 | 状态 |
|------|------|------|
| 邮件发送服务 | SendGrid / AWS SES | 待接入 |
| Google OAuth | Cloud 项目 + Client ID（审核 3-5 天） | 待申请 |
| Facebook Login | 开发者 App ID（审核 5-10 天） | 待申请 |
| 后端 Auth API | 6 个接口 | 待开发 |

### 约束

- 验证码：6 位数字，5 分钟有效，60s 重发，24h 上限 10 次
- 底部必须展示服务条款和隐私政策链接
- OAuth 按钮遵循品牌规范
- 用户数据符合 GDPR

## 八、风险与缓解

| 风险 | 概率 | 缓解 |
|------|------|------|
| OAuth 审核延迟 | 中 | 提前申请，邮箱验证码先上线 |
| 验证码进垃圾箱 | 中 | 高信誉服务商 + SPF/DKIM |
| Token 安全 | 低 | HTTPS + HttpOnly + CSRF |
| 高并发延迟 | 低 | 消息队列 + 限流 |
